========================================
COMO FAZER COMMIT DAS CORREÇÕES
========================================

O Git está detectando o diretório home como repositório.
Execute os comandos abaixo NO DIRETÓRIO DO PROJETO.

OPÇÃO 1: Usar o script batch
-----------------------------
1. Abra o arquivo FAZER_COMMIT.bat (duplo clique)
2. O script vai fazer tudo automaticamente

OPÇÃO 2: Fazer manualmente no terminal
---------------------------------------
1. Abra o PowerShell ou CMD
2. Navegue para o diretório do projeto:
   cd "C:\Users\lmonc\Desktop\MonPO-Monitor de Plano Orçamentario\Monpec_GestaoRural"

3. Verifique se está no diretório correto:
   dir manage.py

4. Adicione os arquivos modificados:
   git add gestao_rural/views.py
   git add gestao_rural/views_demo_setup.py
   git add INSTRUCOES_DOCKERFILE.txt

5. Verifique o que será commitado:
   git status

6. Faça o commit:
   git commit -m "Fix: Resolver problemas de deploy - simplificar demo, criar propriedade automaticamente e redirecionar para landing page"

7. Faça o push:
   git push origin main

OPÇÃO 3: Usar Git GUI ou VS Code
---------------------------------
1. Abra o projeto no VS Code ou Git GUI
2. Você verá os arquivos modificados:
   - gestao_rural/views.py
   - gestao_rural/views_demo_setup.py
   - INSTRUCOES_DOCKERFILE.txt

3. Adicione os arquivos ao stage
4. Faça commit com a mensagem:
   "Fix: Resolver problemas de deploy - simplificar demo, criar propriedade automaticamente e redirecionar para landing page"

5. Faça push para origin main

========================================
IMPORTANTE: CRIAR O DOCKERFILE
========================================

Antes de fazer deploy, você PRECISA criar o Dockerfile manualmente:

1. Crie um arquivo chamado "Dockerfile" (sem extensão) na raiz do projeto
2. Veja o conteúdo em INSTRUCOES_DOCKERFILE.txt
3. Cole o conteúdo no Dockerfile
4. Salve o arquivo
5. Adicione ao git: git add Dockerfile
6. Faça commit: git commit -m "Add: Dockerfile com collectstatic"
7. Faça push: git push origin main
